# V1.2 — 数据治理版 · 任务书

## 1. 版本目标

数据快照、类别映射重排、标签修订追踪、数据质量分析。训练任务绑定不可变快照而非实时数据集。

## 2. 依赖

V1.1 全部验收通过。

## 3. 边界定义

**本版本包含**：
- 数据快照创建/查看（冻结样本清单 + 划分 + 类别体系）
- 训练任务改为绑定快照
- 类别映射重排（class_id 重新排序、类别合并）
- 映射预览 + 标签批量重写
- 标签修订记录（before/after 快照）
- 数据质量统计面板

**本版本不包含**：
- 快照对比 → 后续增强
- 错误样本回流统计 → V4.0
- 多来源数据融合 → 后续

### 任务 1.2.2 — 数据快照服务

**交付物**：`src/features/dataset/SnapshotService.h/.cpp`

```cpp
struct SnapshotInfo {
    QString id;
    QString datasetId;
    QString name;
    int sampleCount;
    int trainCount;
    int valCount;
    int taxonomyVersion;
    QDateTime createdAt;
};

class SnapshotService : public QObject {
    Q_OBJECT
public:
    // 创建快照：冻结当前数据集状态
    // splitRatio: train/val 划分比例
    Q_INVOKABLE QString createSnapshot(const QString &datasetId,
                                        const QString &name,
                                        float splitRatio, int seed = 42);

    // 获取快照列表
    Q_INVOKABLE QList<SnapshotInfo> listSnapshots(const QString &datasetId);

    // 获取快照详情（含样本清单和划分信息）
    Q_INVOKABLE QJsonObject getSnapshotDetail(const QString &snapshotId);

    // 快照是否可变 → 永远不可变
    // 新增样本或修改标签后必须创建新快照
};
```

**快照创建逻辑**：
1. 收集当前 dataset_samples 的所有 sample_id
2. 按 splitRatio 随机划分 train/val
3. 记录当前 taxonomy 版本
4. 写入 dataset_snapshots 表
5. 快照一旦创建不可修改，不可删除（除非未被任何训练引用）

struct MappingRule {

class ClassMappingService : public QObject {
    Q_OBJECT
public:
    // 预览映射影响

    // 5. 插入 annotation_revision 记录

    // 获取映射历史
    Q_INVOKABLE QJsonArray getMappingHistory(const QString &datasetId);

## 5. 验收检查单

1. 创建数据快照 → 快照信息写入数据库
2. 快照内容不可修改 → 新增样本后旧快照不受影响
3. 训练任务绑定快照 → training_runs.snapshot_id 正确
4. 类别映射预览 → 影响范围统计准确
5. 执行映射重排 → 标签文件 class_id 全部正确更新
6. 映射后重新读取标签 → 框数据正确
7. annotation_revisions 记录 before/after 差异
8. class_mapping_revisions 记录映射规则
9. 数据质量面板图表渲染正确
