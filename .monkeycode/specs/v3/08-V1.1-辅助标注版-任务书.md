# V1.1 — 辅助标注版 · 任务书

## 1. 版本目标

用已训练的历史模型对新图片批量推理，生成候选框供人工逐一审核确认，从而加速标注效率。

## 2. 依赖

V1.0 全部验收通过。

## 3. 边界定义

**本版本包含**：
- 推理任务创建（选模型版本 + 目标数据集 + 阈值）
- 批量推理执行（调用 Python 后端 Ultralytics 推理 API）
- 候选框叠加显示（虚线框 + 置信度文字）
- 单张逐框审核（确认 / 拒绝 / 编辑）
- 批量按阈值确认高置信框
- 审核后写回标签文件

**本版本不包含**：
- 低置信样本回流队列 → V4.0
- 自动聚类难例 → V4.0
- OBB 辅助标注 → V3.0

### 任务 1.1.2 — Python 推理 Handler

**交付物**：`backend/labeltorch_backend/handlers/inference.py`

**IPC 命令 `inference.run`**：

请求 payload：
```json
{
    "batch_id": "uuid",
    "model_path": "/path/to/best.pt",
    "image_paths": ["/path/img1.jpg", "/path/img2.jpg"],
    "conf_threshold": 0.25,
    "iou_threshold": 0.45,
    "output_path": "/path/to/project/cache/inference/<batch_id>.json"
}
```

响应：
```json
{
    "success": true,
    "result": { "status": "started", "batch_id": "uuid" }
}
```

推理进度事件（每张图完成后发送）：
```json
{
    "event_type": "task.progress",
    "task_id": "uuid",
    "payload": {
        "current": 15,
        "total": 200,
        "sample_path": "/path/img15.jpg",
        "detections": [
            { "class_id": 0, "cx": 0.5, "cy": 0.3, "w": 0.2, "h": 0.1, "score": 0.92 }
        ]
    }
}
```

推理实现：
```python
async def run_inference(model_path, image_paths, conf, iou, progress_callback):
    from ultralytics import YOLO
    model = YOLO(model_path)
    results = []
    for i, img_path in enumerate(image_paths):
        pred = model.predict(img_path, conf=conf, iou=iou, verbose=False)[0]
        detections = []
        for box in pred.boxes:
            detections.append({
                "class_id": int(box.cls[0]),
                "cx": float(box.xywhn[0][0]),
                "cy": float(box.xywhn[0][1]),
                "w":  float(box.xywhn[0][2]),
                "h":  float(box.xywhn[0][3]),
                "score": float(box.conf[0])
            })
        await progress_callback(i + 1, len(image_paths), img_path, detections)
    return results
```

### 任务 1.1.3 — 候选框审核服务

**交付物**：`src/features/inference/AssistedLabelService.h/.cpp`

```cpp
class AssistedLabelService : public QObject {
    Q_OBJECT
public:
    // 创建推理批次
    Q_INVOKABLE QString createBatch(const QString &modelVersionId,

    void batchStatusChanged(const QString &batchId, const QString &status);
```

### 任务 1.1.4 — AnnotationCanvas 扩展

在 V1.0 的 AnnotationCanvas 基础上，增加辅助标注候选框的渲染支持：

```cpp
// 新增接口
Q_INVOKABLE void loadCandidates(const QJsonArray &candidates);
Q_INVOKABLE void confirmSelectedCandidate();
Q_INVOKABLE void rejectSelectedCandidate();
Q_INVOKABLE int pendingCandidateCount() const;
```

**渲染规则（候选框）**：
- 候选框用**虚线边框**绘制，颜色比已确认框更透明
- 框左上角显示"[类别名] [置信度%]"，如 "[缺陷A] 92%"
- `pending` 状态：黄色虚线
- `confirmed` 状态：同正常框样式（实线）
- `rejected` 状态：灰色，半透明（可选择隐藏）
- `edited` 状态：蓝色实线（表示用户修改过）

## 5. 验收检查单

1. 选择模型版本 + 数据集 → 点击"开始推理" → 推理正常执行
2. 推理进度条正确更新
3. 推理完成后，候选框叠加显示在画布上（虚线黄色）
4. 点击"确认"→ 框变为实线 + 写回标签文件
5. 点击"拒绝"→ 框变灰 + 不写入标签文件
6. 批量确认 ≥ 0.8 → 对应数量的框被确认
7. 审核后的标签文件格式正确（YOLO txt）
8. 审核完成后可立即进行增量训练（V0.6 验收）
