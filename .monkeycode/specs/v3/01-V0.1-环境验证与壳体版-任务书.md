## 1. 版本目标

确认全部开发环境可用，主程序能编译运行不闪退，显示一个完整的主窗口壳体，Python 后端子进程生命周期正常，IPC 通信链路畅通。

用户需提前安装以下软件，详见配套的《开发环境搭建手册》：

| 软件 | 版本 | 安装路径建议 |

| Visual Studio 2022 | 最新 | 默认 |
| Qt 6.11.0 (MSVC 2022 64-bit) | 6.11.0 | F:/A/Qt |
| CMake | 3.21+ | F:/A/CMake |
| Ninja | 最新 | 加入 PATH |
| Python (Miniconda) | 3.11 | F:/A/Miniconda3 |
| NVIDIA 驱动 | ≥ 525.60 | 默认 |
| Git | 最新 | 默认 |

conda 环境创建：
```bash
conda create -n labeltorch python=3.11 -y
conda activate labeltorch
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
pip install ultralytics onnxruntime-gpu
```

### 任务 0.1.4 — Python 后端进程管理

**交付物**：`src/core/PythonBackend.h/.cpp`

**C++ 接口**：

```cpp
class PythonBackend : public QObject {
    Q_OBJECT
    Q_PROPERTY(bool running READ isRunning NOTIFY runningChanged)
    Q_PROPERTY(QString status READ status NOTIFY statusChanged)

public:
    explicit PythonBackend(QObject *parent = nullptr);
    ~PythonBackend();

    bool isRunning() const;
    QString status() const;

    // 启动 Python 后端子进程
    // pythonPath: conda 环境中的 python 可执行文件路径
    Q_INVOKABLE bool start(const QString &pythonPath);

    // 优雅关闭
    Q_INVOKABLE void shutdown();

    // 发送 IPC 请求
    // 返回 request_id，响应通过 responseReceived 信号返回
    Q_INVOKABLE QString sendRequest(const QString &command, const QJsonObject &payload);

signals:
    void runningChanged();
    void statusChanged();
    void responseReceived(const QString &requestId, bool success,
                          const QJsonObject &result, const QJsonObject &error);
    void eventReceived(const QString &eventType, const QString &taskId,
                       const QJsonObject &payload);
    void processError(const QString &errorMessage);

private:
    QProcess *m_process = nullptr;
    // ... 内部实现
};
```

**行为规则**：
1. 使用 `QProcess` 启动 `python -m labeltorch_backend`
2. stdin 写入 JSON 请求，stdout 读取 JSON 响应/事件
3. stderr 转发到日志面板
4. 主程序退出时自动发送 shutdown 命令并等待子进程退出（超时 5 秒后 kill）
5. 子进程异常退出时发出 processError 信号

**验收**：
- 主程序启动后，Python 子进程随之启动
- 主程序退出后，Python 子进程干净退出
- 任务管理器中无残留 python.exe 进程

### 任务 0.1.5 — IPC 握手验证

**交付物**：在 ShellLayout.qml 启动时自动发送 `environment.check`

**Python 端实现**（已有骨架，需完善 handler）：

```python
# backend/labeltorch_backend/handlers/environment.py

async def handle_check(payload: dict) -> dict:
    """检查运行环境"""
    import torch
    import ultralytics
    import platform
    import onnxruntime

    result = {
        "python": platform.python_version(),
        "torch": torch.__version__,
        "torch_cuda": torch.version.cuda or "N/A",
        "cuda_available": torch.cuda.is_available(),
        "gpu_name": torch.cuda.get_device_name(0) if torch.cuda.is_available() else "N/A",
        "gpu_count": torch.cuda.device_count(),
        "ultralytics": ultralytics.__version__,
        "onnxruntime": onnxruntime.__version__,
        "platform": platform.platform(),
    }
    return result
```

**Qt 端行为**：
1. 收到响应后更新顶部状态栏的 GPU 状态指示器
2. 如果 `cuda_available = false`，状态栏显示黄色警告图标
3. 响应内容写入底部日志面板

**验收**：
- 启动后状态栏显示"GPU: RTX xxxx (CUDA 12.1)"
- 日志面板显示完整环境信息
