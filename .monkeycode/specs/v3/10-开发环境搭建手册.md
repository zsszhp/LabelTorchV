# 开发环境搭建手册

> 本文档供用户手动安装开发环境。所有软件建议安装到 F: 盘。

## 1. 必装软件清单

| 序号 | 软件 | 版本 | 建议安装路径 | 状态 |
|---|---|---|---|---|
| 1 | Visual Studio 2022 | 最新 Community | 默认 | 需确认 |
| 2 | Qt 6.11.0 | MSVC 2022 64-bit | F:/A/Qt | 需确认 |
| 3 | CMake | 3.21+ | F:/A/CMake | 需确认 |
| 4 | Ninja | 最新 | F:/A/Ninja（加入 PATH） | 需确认 |
| 5 | Anaconda / Miniconda | Python 3.11 | 用户已有 Anaconda | ✅ 已有 |
| 6 | Git | 最新 | 默认 | 需确认 |
| 7 | NVIDIA 驱动 | ≥ 525.60 | 默认 | 需确认 |

## 2. Anaconda vs Miniconda 选择建议

**结论：你已有 Anaconda，不需要再装 Miniconda。直接用 Anaconda 即可。**

| 维度 | Anaconda | Miniconda |
|---|---|---|
| 安装体积 | ~4.5 GB（预装 250+ 科学计算包） | ~400 MB（仅 conda + Python） |
| 适用场景 | 数据科学全家桶，预装 numpy/pandas/jupyter 等 | 极简主义，按需安装 |
| conda 功能 | 完全相同 | 完全相同 |
| 环境管理 | 完全相同 | 完全相同 |
| 与本项目关系 | 只用 conda 创建独立环境，预装包不影响 | 同 |

**为什么不需要换**：
1. conda 的环境隔离机制使得 Anaconda 和 Miniconda 在功能上完全等价
2. `conda create -n labeltorch python=3.11` 创建的独立环境不受 base 环境影响
3. 唯一区别是磁盘占用，对你无影响

## 3. Visual Studio 2022 安装

下载：https://visualstudio.microsoft.com/

安装时勾选以下工作负载：
- ☑ **使用 C++ 的桌面开发**
  - 确保包含 MSVC v143 x64/x86 编译器
  - 确保包含 Windows 11 SDK（10.0.22621.0 或更高）

验证：
```powershell
# 打开 "x64 Native Tools Command Prompt for VS 2022"
cl.exe   # 应输出 MSVC 编译器版本信息
```

## 4. Qt 6.11.0 安装

下载 Qt Online Installer：https://www.qt.io/download-qt-installer

**步骤**：
1. 运行安装器 → 登录 Qt 账号（免费注册）
2. 安装路径选择 `F:/A/Qt`
3. 在组件选择页面，展开 Qt 6.11.0，勾选：
   - ☑ MSVC 2022 64-bit
   - ☑ Sources（可选，调试时有用）
4. 在 "Additional Libraries" 中勾选：
   - ☑ Qt Quick / Qt QML
   - ☑ Qt SQL
   - ☑ Qt Charts（用于训练曲线图表）
   - ☑ Qt Quick Controls
   - ☑ Qt Quick Dialogs
5. 点击安装

**安装后设置环境变量**（系统环境变量 → 编辑）：

| 变量名 | 值 |
|---|---|
| `Qt6_DIR` | `F:/A/Qt/6.11.0/msvc2022_64` |
| `CMAKE_PREFIX_PATH` | `F:/A/Qt/6.11.0/msvc2022_64` |
| `PATH`（追加） | `F:/A/Qt/6.11.0/msvc2022_64/bin` |

验证：
```powershell
qmake --version   # 应输出 Qt 6.11.0
```

## 5. CMake + Ninja 安装

### CMake
下载：https://cmake.org/download/（选择 Windows x64 Installer）

1. 安装到 `F:/A/CMake`
2. 安装时勾选 **"Add CMake to the system PATH"**

### Ninja
下载：https://github.com/ninja-build/ninja/releases（选择 ninja-win.zip）

1. 解压到 `F:/A/Ninja/`
2. 将 `F:/A/Ninja` 加入系统 PATH

验证：
```powershell
cmake --version   # 应输出 3.21+
ninja --version   # 应输出版本号
```

## 6. Python 环境配置

使用你已有的 Anaconda，打开 Anaconda Prompt：

```bash
# 步骤 1：创建项目专用环境
conda create -n labeltorch python=3.11 -y

# 步骤 2：激活环境
conda activate labeltorch

# 步骤 3：安装 PyTorch + CUDA 12.1
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121

# 步骤 4：安装 Ultralytics YOLO
pip install ultralytics

# 步骤 5：安装 ONNX Runtime GPU
pip install onnxruntime-gpu

# 步骤 6：验证安装
python -c "import torch; print(f'PyTorch {torch.__version__}')"
python -c "print(f'CUDA available: {torch.cuda.is_available()}')"
python -c "print(f'GPU: {torch.cuda.get_device_name(0)}')"
python -c "from ultralytics import YOLO; print('Ultralytics OK')"
python -c "import onnxruntime; print(f'ORT {onnxruntime.__version__}')"
```

**预期输出**：
```
PyTorch 2.x.x
CUDA available: True
GPU: NVIDIA GeForce RTX xxxx
Ultralytics OK
ORT 1.x.x
```

如果 `CUDA available: False`，见下方 FAQ。

## 7. NVIDIA 驱动检查

```powershell
nvidia-smi
```

检查输出：
- 右上角 `CUDA Version` 应 ≥ 12.1
- 驱动版本应 ≥ 525.60
- 应正确识别你的 GPU 型号

如果驱动版本过旧，前往 https://www.nvidia.com/drivers 更新。


## 10. 常见问题（FAQ）

### Q: cmake 报错找不到 Qt6

确认环境变量 `CMAKE_PREFIX_PATH` 指向正确的 Qt 安装路径：
```powershell
echo $env:CMAKE_PREFIX_PATH
# 应输出 F:/A/Qt/6.11.0/msvc2022_64
```

### Q: torch.cuda.is_available() 返回 False

1. 确认 `nvidia-smi` 能正常输出
2. 确认安装的是 CUDA 12.1 版本的 PyTorch：
   ```bash
   pip uninstall torch torchvision
   pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
   ```
3. 确认在 labeltorch 环境中而非 base 环境

### Q: QML 模块加载失败

确认 Qt 安装时勾选了：
- Qt Quick Controls
- Qt QML
- Qt Quick Dialogs

### Q: Ninja 找不到

确认 `F:/A/Ninja` 已加入系统 PATH 环境变量，重启终端后生效。

### Q: Anaconda Prompt 中 conda activate 报错

运行 `conda init powershell` 初始化 conda shell 集成，然后重启终端。
