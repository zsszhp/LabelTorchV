# V4.0 — 主动学习与多任务版 · 任务书

## 1. 版本目标

基于训练结果自动识别难例和低置信样本，构建优先审核队列，加速标注-训练迭代效率。同时扩展支持分类和异常检测任务。

## 2. 依赖

V3.0 全部验收通过。

## 3. 边界定义

**本版本包含**：
- 低置信样本回流队列
- 漏检/误检样本池
- 难例优先审核队列
- 分类任务工作台（基础）
- 异常检测任务工作台（基础）

**本版本不包含**：
- 完整的分类模型训练 pipeline → 后续
- 语义分割 → 后续
- 关键点检测 → 后续

### 任务 4.0.2 — 主动学习分析服务

**交付物**：`backend/labeltorch_backend/handlers/active_learning.py`

**IPC 命令 `active_learning.analyze`**：

请求：
```json
{
    "command": "active_learning.analyze",
    "payload": {
        "model_version_id": "uuid",
        "dataset_id": "uuid",
        "analysis_types": ["low_confidence", "false_positive"],
        "low_conf_range": [0.3, 0.7],
        "iou_threshold": 0.5
    }
}
```

**分析逻辑**：

1. **低置信样本**（low_confidence）：
   - 用指定模型对数据集推理
   - 筛选置信度在 `[min_conf, max_conf]` 区间的检测框
   - 按置信度从低到高排序
   - 对应样本加入审核队列

2. **误检样本**（false_positive）：
   - 推理结果中检测到但标签中不存在（IoU < threshold）的框
   - 可能是模型误检，也可能是漏标

3. **漏检样本**（false_negative）：
   - 标签中有但推理未检出（IoU < threshold）的框
   - 可能是训练不足或标注有误

4. **难例样本**（hard_case）：
   - 综合得分：低置信 + 高 loss（如果有记录）
   - 按综合难度排序

## 5. 验收检查单

1. 主动学习分析完成 → 4 类队列样本数正确
2. 低置信队列 → 按置信度排序，审核后状态更新
3. 误检队列 → 显示推理框和标签框对比
4. 审核完成 → 可创建新快照 → 增量训练
5. 分类任务 → 图片分类标注 → 保存 → 目录结构正确
6. 分类训练 → 使用 Ultralytics classify 模式 → 1 epoch 冒烟通过
