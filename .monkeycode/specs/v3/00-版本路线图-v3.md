# 标炬（LabelTorch）版本路线图 v3

> 本文件是 v3 规划体系的总纲。所有版本按序实现，每个版本可独立测试验收。

## 技术决策摘要

| 项 | 决策 |
|---|---|
| 前端 | Qt 6.11 + QML + C++17 |
| 后端 | Python 3.11 + Ultralytics |
| IPC | stdin/stdout JSON-RPC |
| 数据库 | SQLite 3（WAL 模式）|
| 构建 | CMake 3.21+ / Ninja / MSVC 2022 |
| 打包 | Qt IFW + windeployqt + embed Python |
| CUDA | 12.1（PyTorch wheel 自带运行时）|
| UI 语言 | 仅中文 |
| 模型优先级 | YOLOv8 > v11 > v10 > v5 > v8-obb |

## V0.1 — 环境验证与壳体版

**目标**：确认全部开发环境可用，主程序能编译运行不闪退，显示一个完整的主窗口壳体。

**功能清单**：
1. 开发环境搭建文档 + 验证脚本
2. CMake 构建通过，生成可执行文件
3. Python 后端进程启动/关闭生命周期管理
4. IPC 通信握手验证（environment.check 命令往返）
5. 主窗口壳体：左侧导航栏 + 顶部状态栏 + 中央工作区 + 底部日志面板
6. SQLite 数据库初始化（建表脚本）
7. 全局主题系统（深色主题 + 工业风视觉风格）

**依赖**：无

**验收标准**：
- [ ] cmake 配置 + 构建零错误
- [ ] 启动主程序，窗口正常显示，不闪退
- [ ] 左侧导航可点击切换页面占位符
- [ ] Python 后端子进程随主程序启动/退出
- [ ] `environment.check` 命令返回 Python/CUDA/Ultralytics 版本信息
- [ ] SQLite 数据库文件可创建，表结构正确

## V0.3 — 数据导入版

**目标**：能导入 YOLO txt 格式数据集，浏览图片缩略图，查看类别列表。

**功能清单**：
1. 数据导入页面：选择图片目录 + 标签目录
2. 导入扫描：匹配图片-标签对，记录异常（缺标签、空标签、格式错误）
3. 导入进度条
4. 导入报告页（成功数/失败数/异常列表）
5. 样本缩略图网格浏览
6. 类别列表展示（从标签文件自动提取）
7. 类别名称编辑（给 class_id 分配可读名称）
8. 数据集统计面板（样本数、类别分布柱状图、图片尺寸分布）
9. 数据集记录写入 SQLite

**依赖**：V0.2

**验收标准**：
- [ ] 选择包含 100+ 图片的 YOLO 目录，导入成功率 100%
- [ ] 异常标签文件被正确标记，不阻断导入
- [ ] 缩略图网格加载流畅，支持滚动
- [ ] 类别列表与标签文件中的 class_id 一致
- [ ] 数据统计图表正确渲染

## V0.4 — 训练闭环版 ⭐ 核心里程碑

**目标**：能用 YOLOv8 完成一次完整的训练闭环：配置参数 → 启动训练 → 查看日志 → 训练完成。

**功能清单**：
1. 训练工作台页面
2. 模型选择器（当前仅 YOLOv8，下拉选择变体：n/s/m/l/x）
3. 训练参数面板（img_size、batch、epochs、patience、device、workers）
4. 数据集选择（从已导入的数据集中选择）
5. train/val 自动划分（可配比例，默认 80/20）
6. 训练启动 → Python 后端调用 Ultralytics YOLO API
7. 实时训练日志流（底部日志面板）
8. 训练进度展示（当前 epoch / 总 epoch、loss 曲线图）
9. 训练状态管理（running/stopping/succeeded/failed/cancelled）
10. 训练完成后自动注册 ModelVersion 到数据库
11. 早停支持（patience 到达后自动停止）
12. 训练结果摘要页（mAP50、mAP50-95、precision、recall）

**依赖**：V0.3

**验收标准**：
- [ ] 选择数据集 + 设置参数 + 点击训练，训练正常启动
- [ ] 日志面板实时刷新训练输出
- [ ] loss 曲线图随 epoch 实时更新
- [ ] 训练完成后，best.pt 权重文件存在于项目目录
- [ ] ModelVersion 记录写入数据库，含 mAP 等指标
- [ ] 点击停止按钮，训练可优雅终止
- [ ] 训练中关闭主程序，Python 后端子进程不残留

## V0.6 — 增量训练版

**目标**：能选择历史模型权重作为初始化权重继续训练，形成模型谱系链。

**功能清单**：
1. 训练参数面板增加"预训练权重"选择器
2. 三种训练模式 UI 明确区分：从零训练 / 预训练权重 / 增量训练
3. 增量训练时 parent_model_version_id 自动记录
4. 模型版本列表增加父子关系展示（谱系链/树形视图）
5. 新增图片到数据集（追加导入）
6. 重新划分 train/val

**依赖**：V0.5

**验收标准**：
- [ ] 选择上一版 best.pt 作为权重启动增量训练
- [ ] 训练完成后 ModelVersion.parent_model_version_id 正确
- [ ] 谱系链视图正确展示父子关系
- [ ] 追加导入新图片后样本数增加，可重新训练

## V1.0 — 标注引擎版（BBox）⭐ 重大里程碑

**目标**：高性能图像画布 + BBox 标注编辑能力。

**功能清单**：
1. 高性能图像画布（C++ QQuickItem，GPU 加速渲染）
2. 图像缩放（滚轮）、平移（空格+拖拽）
3. 样本列表侧边栏 + 前后切换（A/D 键）
4. BBox 创建（拖拽画框）
5. BBox 编辑（移动、缩放边角、删除）
6. 类别选择面板 + 快捷键（1-9 键）
7. 标签保存（YOLO txt 格式原子写回）
8. 标注状态提示（已保存/未保存）
9. 标注统计（当前样本框数、总体完成进度）
10. 快捷键体系完整实现

**依赖**：V0.6

**验收标准**：
- [ ] 4000×3000 分辨率图片缩放平移流畅（≤16ms 帧时间）
- [ ] 画框 → 保存 → 重载，框坐标一致
- [ ] 连续切换 100 张样本无卡顿无崩溃
- [ ] 所有快捷键生效

## V1.1 — 辅助标注版

**目标**：用已训练模型对新图片推理，生成候选框供人工审核确认。

**功能清单**：
1. 推理任务创建（选择模型版本 + 目标数据集 + 置信度阈值）
2. 批量推理执行 + 进度
3. 候选框叠加显示（虚线框 + 置信度标签）
4. 单张审核：逐框确认/拒绝/编辑
5. 批量确认（按置信度阈值一键确认高置信框）
6. 确认后的框写回标签文件
7. 审核状态统计（待审核/已确认/已拒绝/已编辑）

**依赖**：V1.0

**验收标准**：
- [ ] 推理结果正确显示在画布上
- [ ] 确认的框写回标签后格式正确
- [ ] 批量确认后增量训练，mAP 有提升
