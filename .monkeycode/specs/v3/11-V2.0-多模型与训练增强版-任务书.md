# V2.0 — 多模型家族与训练增强版 · 任务书

## 1. 版本目标

扩展训练适配器支持 YOLOv5/v10/v11，增加 AMP、断点续训、实验对比功能。

## 2. 依赖

V1.2 全部验收通过。

## 3. 边界定义

**本版本包含**：
- TrainingAdapter 适配器注册机制
- YOLOv5 / YOLOv10 / YOLOv11 适配器
- 模型家族选择器 UI 扩展
- AMP（自动混合精度）
- 断点续训（从 last.pt 恢复）
- 实验对比面板

**本版本不包含**：
- 多卡训练 → 后续
- 超参搜索 → 后续
- YOLOv8-OBB → V3.0

### 任务 2.0.2 — 适配器注册机制

**交付物**：`backend/labeltorch_backend/adapters/registry.py`

```python
class AdapterRegistry:
    """训练适配器注册表"""
    _adapters: dict[str, type[TrainingAdapter]] = {}

    @classmethod
    def register(cls, adapter_class: type[TrainingAdapter]):
        """注册适配器（装饰器方式）"""
        instance = adapter_class()
        for family in instance.supported_families():
            cls._adapters[family] = adapter_class

    @classmethod
    def get(cls, model_family: str) -> TrainingAdapter:
        """获取适配器实例"""
        adapter_class = cls._adapters.get(model_family)
        if not adapter_class:
            raise ValueError(f"不支持的模型家族: {model_family}")
        return adapter_class()

    @classmethod
    def list_families(cls) -> list[str]:
        """列出所有支持的模型家族"""
        return list(cls._adapters.keys())

    @classmethod
    def list_variants(cls, model_family: str) -> list[str]:
        """列出某个家族的所有变体"""
        return cls.get(model_family).list_variants()
```

### 任务 2.0.3 — YOLOv5 适配器

```python
@AdapterRegistry.register
class YOLOv5Adapter(TrainingAdapter):
    def adapter_id(self) -> str:
        return "yolov5"

    def supported_families(self) -> list[str]:
        return ["yolov5"]

    def list_variants(self) -> list[str]:
        return ["yolov5n", "yolov5s", "yolov5m", "yolov5l", "yolov5x"]

    async def start_training(self, config, data_yaml, output_dir, progress_callback):
        from ultralytics import YOLO
        model = YOLO(f"{config.model_variant}.pt")
        results = model.train(
            data=data_yaml,
            imgsz=config.img_size,
            batch=config.batch,
            epochs=config.epochs,
            patience=config.patience,
            device=config.device,
            workers=config.workers,
            amp=config.amp,
            project=output_dir,
            name="train",
            exist_ok=True,
        )
        # ... 解析 results
```

> 注：YOLOv5 通过 Ultralytics 统一 API 调用，不使用 YOLOv5 原生仓库。

### 任务 2.0.5 — 模型家族选择器扩展

**修改**：`src/features/training/TrainingPage.qml`

模型选择区域增加动态查询：
1. 启动时发送 `adapter.list_families` 命令获取所有支持的模型家族
2. 用户选择家族后发送 `adapter.list_variants` 获取变体列表
3. 下拉框动态填充

新增 IPC 命令：

```json
// adapter.list_families
{ "command": "adapter.list_families", "payload": {} }
// 返回：{ "result": ["yolov5", "yolov8", "yolov10", "yolo11"] }

// adapter.list_variants
{ "command": "adapter.list_variants", "payload": { "family": "yolov8" } }
// 返回：{ "result": ["yolov8n", "yolov8s", "yolov8m", "yolov8l", "yolov8x"] }
```

### 任务 2.0.6 — AMP 与断点续训

**AMP**：
- 训练参数面板增加"混合精度"开关（默认关闭）
- 开启后 `config.amp = True` 传入适配器

**断点续训**：
- 训练状态为 `failed` 或 `cancelled` 时，版本列表显示"续训"按钮
- 点击"续训" → 查找 last.pt → 设置 `resume=True` → 启动新训练
- 新 training_run 的 `resume_from` 记录原 run_id
- Ultralytics API：`model.train(resume=True)`

## 5. 验收检查单

1. `adapter.list_families` 返回 4 个模型家族
2. 选择 yolov5n → 训练 1 epoch → 成功
3. 选择 yolov10n → 训练 1 epoch → 成功
4. 选择 yolo11n → 训练 1 epoch → 成功
5. AMP 开启后训练速度有提升（对比关闭时）
6. 训练中途取消 → 点击"续训" → 从 last.pt 恢复 → epoch 编号连续
7. 选择 3 个版本对比 → 指标表格数据正确
8. mAP 趋势图多曲线叠加正确
