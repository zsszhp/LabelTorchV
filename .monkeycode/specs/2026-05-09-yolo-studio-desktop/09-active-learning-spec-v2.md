# 主动学习与难例回流规范 v2

## 1. 目标

主动学习系统用于持续提升训练效果，而不是仅提升流程便利性。它负责发现最值得重新标注或复审的样本，并推动增量数据闭环。

## 2. 回流样本来源

1. 低置信候选框样本。
2. 高冲突类别样本。
3. 漏检样本。
4. 误检样本。
5. 人工重点标记的难例。

## 3. 队列设计

### 3.1 队列类型

1. `low-confidence-queue`
2. `false-positive-queue`
3. `false-negative-queue`
4. `hard-case-queue`

### 3.2 排序策略

优先级从高到低建议为：

1. 漏检高风险样本
2. 高价值类别低置信样本
3. 重复误检样本
4. 长尾类别稀缺样本

## 4. MVP 与后续边界

### 4.1 MVP

1. 仅实现历史模型辅助标注。
2. 仅实现置信度阈值筛选。
3. 仅实现批量确认与人工修订闭环。

### 4.2 中期版本

1. 自动收集低置信样本池。
2. 支持按类别查看候选样本。

### 4.3 成熟版本

1. 自动聚类难例。
2. 自动生成下一轮增量标注建议集。
3. 自动生成训练迭代建议。

## 5. 与数据治理耦合要求

1. 回流队列中的每个样本必须关联来源模型版本。
2. 样本完成修订后必须形成新的标签修订记录。
3. 进入下一轮训练时必须形成新的数据快照。
