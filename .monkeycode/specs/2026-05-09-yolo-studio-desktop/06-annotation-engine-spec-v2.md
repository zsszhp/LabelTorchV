# 标注引擎规范 v2

## 1. 目标

标注引擎负责高性能图像浏览、几何编辑、审核与标签落盘。第一阶段 UI 仅开放 bbox，但底层几何模型必须一次性支持向 OBB 和 Polygon 扩展。

## 2. 设计原则

1. 底层几何统一，UI 功能分阶段开放。
2. 图像浏览与标注渲染解耦。
3. 写回安全优先于保存速度。
4. 交互响应优先级高于非关键后台任务。

## 3. 几何模型

### 3.1 基础图元

1. `AxisAlignedBox`
2. `RotatedBox`
3. `Polygon`
4. `PointHandle`

### 3.2 统一属性

1. `id`
2. `class_id`
3. `class_name`
4. `score`
5. `source_type`
6. `is_confirmed`
7. `is_selected`
8. `z_index`

### 3.3 坐标系

1. 图像坐标系：以原始像素为准。
2. 视口坐标系：以当前缩放后的显示区域为准。
3. 场景坐标系：用于渲染和命中测试。

## 4. 渲染架构

### 4.1 分层

1. 背景图层：原始图像。
2. 标注图层：框、多边形、标签文本。
3. 交互图层：控制点、吸附线、框选区域。
4. 覆盖图层：辅助标注置信度、审核状态、错误提示。

### 4.2 性能要求

1. 高分辨率图像缩放时优先平滑交互，再逐步提升清晰度。
2. 多框场景仅局部重绘变化区域。
3. 样本切换时预加载相邻样本缩略图和标签。

## 5. 交互模型

### 5.1 MVP 必须支持

1. 新增 bbox。
2. 删除 bbox。
3. 拖拽移动 bbox。
4. 拖拽边角缩放 bbox。
5. 修改类别。
6. 批量确认辅助标注结果。
7. 单样本保存。

### 5.2 后续支持

1. OBB 旋转。
2. Polygon 顶点编辑。
3. 批量框类别替换。
4. 撤销/重做。

### 5.3 快捷键体系

1. `W` 新建框
2. `Delete` 删除所选
3. `Ctrl+S` 保存
4. `A/D` 切换前后样本
5. `1..9` 快速切换类别
6. `Space` 拖动画布

## 6. 辅助标注审核规则

1. 辅助框必须带来源模型版本与置信度。
2. 辅助框分为 `pending`、`confirmed`、`rejected`、`edited` 四种状态。
3. 用户编辑过的辅助框必须将来源状态改为 `edited`。

## 7. 标签读写规则

1. 读取时保留原始顺序和解析异常信息。
2. 写回时统一按项目类别体系输出。
3. 写回先生成临时文件，再原子替换正式标签文件。
4. 写回完成后立即校验文件可再次解析。

## 8. 验收标准

1. 单样本反复加载与保存不丢框。
2. 类别切换后重载结果一致。
3. 非法框不会写入标签。
4. 样本连续切换时画布不闪退、不明显卡顿。
