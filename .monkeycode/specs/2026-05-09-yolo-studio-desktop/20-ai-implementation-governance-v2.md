# AI 实施治理规范 v2

## 1. 目标

确保后续参与实现的 AI 严格遵守本规划，不擅自简化、越界或改变架构决策。

## 2. 实施前置要求

1. 必须先阅读 `00` 到 `18` 全部 v2 核心规划文档。
2. 必须先完成环境检查。
3. 必须声明本轮实现的 Phase 和子目标。
4. 按需阅读 `21-reference-links.md` 中的参考项目，借鉴技术路线与管理方式。

## 3. 实施边界

1. 不得擅自把混合架构改回纯 Python。
2. 不得绕过数据快照直接训练。
3. 不得把辅助标注做成只读浏览功能。
4. 不得省略双平台 Git 备份与中文提交说明。

## 4. 提交要求

1. 每个小阶段完成后提交一次。
2. 每个大阶段完成后再做一次总提交。
3. 所有提交说明必须用中文，且内容清晰完整。

## 5. 测试要求

1. 每完成一个功能必须做详细测试。
2. 每完成一个大功能必须做整体验证。
3. 未测试通过不得进入下一阶段。

## 6. 发布要求

1. 全部任务完成后必须补全中英文 README。
2. 成熟版必须打包、发布、创建 Release。
