# 错误恢复与回滚规范 v2

## 1. 目标

保证关键数据、训练与导出流程在失败时可恢复、可复盘、可回滚。

## 2. 必须支持回滚的操作

1. 类别映射重写。
2. 标签批量写回。
3. 数据划分重建。
4. 模型版本标记切换。

## 3. 恢复策略

### 3.1 标签写回

1. 先写临时文件。
2. 再执行原子替换。
3. 失败时保留旧文件与错误日志。

### 3.2 类别映射

1. 执行前生成影响预览。
2. 执行后记录 mapping revision。
3. 若校验失败，整体回滚。

### 3.3 训练任务

1. 失败时保留日志、配置、部分产物路径。
2. 不清理失败上下文。

## 4. 崩溃恢复

1. 主程序重启后应恢复最近项目上下文。
2. 若存在未完成任务，需显示恢复或标记失败。
3. 若存在未保存标注，需显示恢复提示。
