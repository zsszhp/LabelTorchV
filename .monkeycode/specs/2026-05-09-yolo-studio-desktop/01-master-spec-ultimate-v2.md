# 标炬（LabelTorch）终局总规范 v2

## 1. 总述

本规范是标炬项目的最高级别规划文档。其他所有文档都必须服从本规范定义的产品定位、架构原则、数据治理要求、质量红线和版本路线。

## 2. 核心系统边界

标炬由以下 8 个一级子系统构成：

1. Desktop Shell：桌面应用壳、导航、全局状态、资源监控、任务提示。
2. Project and Dataset Governance：项目、数据集、类别体系、数据快照、标签修订治理。
3. Annotation Engine：图像浏览、bbox/obb/polygon 几何编辑与审核交互。
4. Training Orchestrator：训练任务编排、配置校验、日志采集、状态流转。
5. Inference and Assisted Labeling：历史模型推理、候选框筛选、批量确认。
6. Model Registry and Evaluation：模型版本、实验快照、指标对比、导出物管理。
7. Release and Packaging：绿色包、CPU/CUDA 双发包、Release 管理。
8. Quality and Recovery：测试矩阵、崩溃恢复、原子写入、回滚策略。

## 3. 产品级质量红线

### 3.1 数据红线

1. 不允许未经确认直接覆盖原始标签。
2. 不允许类别映射重写后无法追溯映射历史。
3. 不允许训练完成后无法定位训练所用数据快照。

### 3.2 训练红线

1. 不允许训练配置无快照。
2. 不允许模型版本无评估指标。
3. 不允许导出物无来源版本信息。

### 3.3 稳定性红线

1. 不允许训练子进程崩溃导致主界面退出。
2. 不允许标签写回部分成功、部分失败且无告警。
3. 不允许发布包无法自检运行环境。

## 4. 最优技术路线结论

### 4.1 最终建议

采用混合架构：

1. `Qt 6 + QML + C++` 负责桌面主程序、标注引擎、几何与渲染性能核心。
2. `Python 3.11+` 负责 Ultralytics 训练后端、推理、导出、模型生态适配。
3. `SQLite` 负责元数据。
4. `本地文件系统` 负责图片、标签、模型产物、导出产物与缓存。
5. `本地 IPC` 负责桌面主程序与 Python 后端解耦。

### 4.2 为什么不是纯 Python

纯 Python 方案适合作为原型和中期产品方案，但在以下方面不是终局最优：

1. 高分辨率画布性能上限更低。
2. 重图像交互场景中长期稳定性与原生体验更弱。
3. OBB/Polygon 扩展时底层几何和渲染优化空间不足。

### 4.3 为什么不是纯 C++

纯 C++ 方案在桌面层强，但在训练生态与 YOLO 适配上并非最优：

1. Ultralytics 与大量模型工具链天然偏 Python。
2. 后续支持更多训练器时 Python 生态更成熟。
3. 导出与推理验证流程更适合保留 Python 侧灵活性。

### 4.4 为什么不是 Electron 或 Tauri

不采用 Web 技术作为主桌面前端的原因：

1. 本项目属于重图像、重交互、重本地资源控制软件。
2. 图像标注与大图浏览对原生图形与事件响应更敏感。
3. Web 技术可用于管理型软件，但不是工业级标注工作台的最优终局方案。

## 5. 终局能力地图

### 5.1 数据治理能力

1. 数据集导入。
2. 数据集校验。
3. 类别体系管理。

1. 多 YOLO 家族统一编排。

1. 低置信样本回流。
2. 漏检误检样本池。
3. 难例优先审核队列。

### 5.5 开源与发布能力

1. 中英文 README。
2. CPU/CUDA 双发布包。

4. 可复现安装与构建说明。

## 6. 数据、模型、实验三层版本体系

### 6.1 数据版本

数据版本用于描述某一时刻的样本集合、数据划分和标签状态。

应包含：

1. 样本列表。
2. 类别体系版本。
3. 标签修订版本。
4. train/val/test 划分结果。

### 6.2 模型版本

模型版本用于描述训练产物与其来源。

应包含：

1. 训练任务 id。
2. 父模型版本 id。
3. 最佳权重路径。
4. 导出产物清单。
5. 指标快照。

### 6.3 实验版本

实验版本用于描述训练配置和运行环境。

应包含：

1. 模型家族。
2. 参数快照。
3. 设备信息。
4. 日志路径。
5. 训练结果与异常信息。

## 7. 成熟产品判定标准

当下列条件同时满足时，可认为标炬达到成熟产品状态：

1. 检测任务闭环稳定可用。
2. OBB 深度支持完成。
3. 主动学习闭环可用。
4. 多任务扩展架构稳定。
5. 双包发布稳定。
6. 环境检查、回滚、Release 流程完整。
